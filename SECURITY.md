# Security Policy

## Supported Versions

Por favor, garanta que a versão do seu sistema e das dependências estejam o mais atualizadas possível. Falhas descobertas por nossos auditores de segurança (SAST/DAST) são mitigadas na `master`.

| Version | Supported          |
| ------- | ------------------ |
| v0.2.x  | :white_check_mark: |
| < 0.2.0 | :x:                |

## Reporting a Vulnerability

A segurança dos casais e cerimonialistas na nossa plataforma (WeddingApp) é nossa maior prioridade. 

Se você for um pesquisador de segurança ou pentester e acreditou encontrar uma vulnerabilidade (como XSS, IDOR, falha de Autenticação/RLS do Supabase, etc), por favor nos relate confidencialmente. 

**Como reportar:**
1. Envie um e-mail com POC (Proof of Concept) e logs de request em anexo para **resper@ness.com.br** ou **resper@bekaa.eu**.
2. Nossa equipe do Blue Team confirmará o recebimento deste e-mail em até 24 horas úteis.
3. Pedimos 48 a 72 horas para fornecer um cronograma de correção antes de qualquer publicidade.

*Não* reporte vulnerabilidades de segurança abrindo uma Issue pública no GitHub. 

Agradecemos imensamente o esforço coletivo para mantermos esta aplicação Cloud segura.
